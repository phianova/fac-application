Why I want to start a career in software development

My background is in welfare policy and advice work. I had never considered a career in tech until the last 2 years, but my last role for a small policy consultancy and software company involved a lot of hands-on collaboration with a team of developers.
My role involved using my policy and legal knowledge to maintain the logic for a software product calculating welfare benefit entitlement for low-income users.
An Excel-based record of this logic was the basis for integration tests for the policy engine underlying the software - meaning I would regularly be working with developers and engineers to debug the engine's Typescript code when integration tests failed or a logic issue was found.
I thoroughly enjoyed the problem-solving aspects of this work and found myself fascinated by the code and wanting to learn how to do more with it.
This role also involved performing data analysis in Stata (an R-based statistical programming language) and SQL, which again piqued my interest in the practical applications of code to what I was working on.
Basically, I was hooked.

Since then I’ve been exploring as many self-paced study avenues as I can to begin to gain the technical skills I will need for a career in software development.
I’m keen to understand underlying ideas and principles in computer science too, so I've started working through the Harvard CS50 course.
I want to fully apply myself to something very different to my previous experiences, and exercise my problem-solving skills while developing a new skillset and understanding new technologies.

Building this site for my Founders and Coders application

I started building this site having just completed the preparation work for the Founders & Coders course.
I didn't really have any prior JavaScript experience, so this was a real hands-on learning experience for me.
I started with the homepage, and wanted to use my newly-gained knowledge of CSS Flexbox to create the linked images below my intro text container.
I learnt how to create fixed headers and footers, and later amended my header to make it a function navigation bar in pages other than the homepage.
I know tiled baby seal cartoons aren't everyone's cup of tea but that background was one of the first things I learnt to do in CSS without referring directly back to the FreeCodeCamp course, so I've kept it in there as a kind of badge of honour.
I knew I needed some unique features to showcase what I'd learnt. I think I saw someone else mentioning on the Founders & Coders Discord channel that they'd built their showcase pieces based on their own interests, and I thought that was a great idea.
I had a few ideas around this and decided to start with a mood-based Spotify playlist, as I'm a music lover.
This took a lot of experimentation and Googling and Stack-Overflow searching to work out, but I was really happy with the end product.

Building Mudlark (the game) for my Founders and Coders application

Again, building the game was a real challenge for me. I had a clear idea of the story and what I wanted the game to look and feel like, but starting from scratch initially felt quite intimidating.
I started by looking a couple of tutorials to get the gist of the standard elements needed to build a game.
I initially followed some of this tutorial to get an idea of how to set up a game environment using HTML Canvas.
I also used this guide to help me understand the capabilities of the canvas and how to go about designing standard game elements.
I later borrowed the idea of a "toggleScreen()" function from this video which I found really helpful as I wanted my game to have a clear story.
As I went along, I decided between different ways of implementing game elements that I had seen, and refined my code to reflect my game's actual needs.
The first thing I did was code the movement of the player and the dog, and determine what happened when they collided.
I then focused on my hidden items, and worked out how to randomly assign both their position and their type.
I then created my starting screens, and used the same method to implement the screens that display when the items are found.
Then, I created the scoring system and determined the conditions to win the game and what happened when that was achieved.
I decided the game needed a tiny bit more of a challenging/competitive element so I added a timer.
If I had had more time, I would have added further levels to the game, with additional dogs to make it more difficult.
I also had planned to try and include a second area which the player could access via the leftmost border of the screen to make it harder to find the hidden items.
The "tide coming in" screen that displays once the game has ended was also in my initial plan! I wanted to have that happen during the game, cutting off some of the game area to make it more difficult to find the items.
I may make these changes in a separate branch while the application is in progress, just for fun! But thought it was better to ensure I had something workable to submit by the deadline.
